Azure — Interview Cram Sheet

Condensed recall of the expert-signal lines & key trade-offs. Verified vs Microsoft Learn, Jul 2026 · part of the Azure Expert Curriculum.
The closing line: “Given the problem and its constraints, I pick services, justify trade-offs across the 5 WAF pillars, and hand exact execution to automation/AI.” Expertise = judgement, not memorised commands.

The 4 mental models

  • Shared responsibility — Azure secures the platform; you secure what's in it.
  • Service ladder — IaaS→PaaS→FaaS→SaaS = less control, less ops.
  • Control vs data plane — ARM creates resources; data plane does the work.
  • Everything is a resource with an ID, governed by RBAC/Policy/tags/locks.

WAF — 5 pillars (narrate every design)

  • Reliability · Security · Cost · Operational Excellence · Performance
  • CAF = the org journey: Strategy→Plan→Ready→Adopt→Govern→Manage.
  • WAF = one workload; CAF = whole org. Name the right altitude.

3 words people confuse

  • Elasticity = auto scale to demand.
  • Scalability = the ceiling you can reach.
  • Resilience = surviving failure. Design each deliberately.

Hierarchy & global

  • Mgmt Group → Subscription → Resource Group → Resource. RBAC/Policy inherit downward.
  • Region = many datacenters; AZ = isolated DC. Design across ≥3 AZs.
  • Pick region for: latency + data residency + service availability.

Identity = the perimeter

  • No secrets in code. Managed Identity → RBAC role → token, never a password.
  • Secrets that must exist → Key Vault.
  • PIM = JIT admin elevation; Conditional Access = Zero-Trust enforcement.

Compute — pick the most managed

  • VM → VMSS → App Service → Functions → Container Apps → AKS → ACI → Batch.
  • Default to most managed; step down to VM only with a real constraint, and justify it.
  • Serverless (Functions, Container Apps) = scale to zero, pay per use.
  • Know when NOT to use AKS — it's an ops commitment.

Networking (the differentiator)

  • LB (L4) → App Gateway (L7+WAF) → Front Door (global+CDN+WAF) → Traffic Mgr (DNS).
  • Private Endpoint = make PaaS (SQL/Storage) non-public.
  • Hub-spoke = shared firewall/DNS/gateway in hub; workloads in spokes.
  • VPN (over internet) vs ExpressRoute (private circuit).

Storage

  • Account = Blob / File / Queue / Table.
  • Tiers: Hot/Cool/Cold/Archive (+ Smart auto-tier). Match to data temperature.
  • Redundancy: LRS→ZRS→GRS/GZRS. Match to RPO/compliance.

Databases

  • SQL DB (single managed) · Managed Instance (lift & shift) · SQL-on-VM (control).
  • Cosmos DB 5 levels: Strong→Bounded→Session→Consistent Prefix→Eventual (CAP/PACELC).
  • OLTP ≠ OLAP — don't run analytics on the transactional store.

Messaging — event vs message

  • Event = something happened (Event Grid / Event Hubs).
  • Message = please do something (Service Bus).
  • Event Hubs = high-volume streaming; APIM = API gateway; Logic Apps = low-code flows.
  • Messaging buys loose coupling + independent scaling.

AI / ML (current naming)

  • Microsoft Foundry (was Azure AI Studio/Foundry) — unified AI+agents PaaS.
  • Azure OpenAI = a Foundry resource; AI Services = Foundry Tools.
  • RAG = Blob/Lake → embeddings in AI Search → grounded LLM → APIM + identity + App Insights.

Security & Zero Trust

  • Verify explicitly · least privilege · assume breach.
  • Defender for Cloud = posture (Secure Score); Sentinel = SIEM/SOAR.
  • Defense in depth: identity→network→host→app→data.

Governance & cost (FinOps)

  • Policy = enforce rules; Landing Zone = governed foundation to deploy into.
  • Levers: PAYG · Reservations · Savings Plans · Spot · Hybrid Benefit.
  • “Right-size from telemetry, reserve the baseline, burst on consumption, tag everything.”

IaC / DevOps

  • Bicep first (compiles to ARM); Terraform multi-cloud.
  • Declarative = desired state, idempotent, reviewable. Portal-clicking doesn't scale.
  • App Service slots = blue-green / zero-downtime.

Observability

  • Metrics (what) · Logs (why) · Traces (where).
  • App Insights (APM) + Log Analytics (KQL) + Alerts/Action Groups.
  • “Can't promise an SLA you can't measure.”

Reliability & DR

  • AZs = DC fault; multi-region = region fault.
  • RTO/RPO define DR — ask the business first.
  • Composite SLA: services in series multiply (two 99.9% ≈ 99.8%); redundancy raises it.
  • Every extra nine costs money — design to the stated SLA, no more.

Certs (proof of breadth)

  • AZ-900 → AZ-104 → AZ-305 (architect; needs AZ-104; gold standard).
  • AZ-204 (dev) · AZ-500 (sec) · AZ-700 (net) · DP-203 (data) · AI-102 (AI).

One reference architecture

  • App Service (zone-redundant) → SQL via Private Endpoint → Key Vault + Managed Identity → Front Door + WAF → App Insights → Bicep + CI/CD → Reserved + Hybrid Benefit.
  • Defend one design across all 5 pillars and you can defend any.

Always close the loop on 5

  • Identity (managed identity, least priv)
  • Network (private, segmented)
  • Reliability (zones, SLA, DR)
  • Observability (monitor + alerts)
  • Cost (right-size, reserve, tag)

.NET-on-Azure signals

  • DefaultAzureCredential: az login locally, MI deployed — same code; pin ManagedIdentityClientId in prod.
  • SQL passwordless: Authentication=Active Directory Default + CREATE USER … FROM EXTERNAL PROVIDER.
  • Functions: isolated worker only (in-process EOL 2026-11-10); Flex Consumption = recommended serverless.
  • Aspire models the distributed app; azd up deploys it to ACA/App Service.

Deployment & delivery

  • Pipelines auth via OIDC federation — zero stored deploy secrets.
  • Zero-downtime: slots (App Service) / revision weights (ACA); feature flags decouple release from deploy.
  • what-if previews; deployment stacks deny drift + clean teardown.
  • DB migrations: pipeline step, expand→contract, never at multi-instance startup.

Data-engineering marks

  • Cosmos: RU/s divide across physical partitions — hot key throttles at "enough" RU/s; design the key (cardinality · spread · query-match).
  • Change feed → materialized views; TransactionalBatch = one logical partition only.
  • Outbox at every DB-write→publish seam; consumers idempotent; DLQ has a runbook.
  • Private endpoint issues are DNS issues: privatelink zone + zone link, nslookup first.

SLA math (do it live)

  • Series multiplies: 99.99×99.95×99.99 ≈ 99.93% (~6 h/yr).
  • Redundancy compensates: two 99.9% paths ≈ 99.9999% for that hop.
  • SLO ≤ composite SLA of the chain — or shorten the chain (queues).
  • Every extra nine costs money: design to the stated RTO/RPO, no more.
